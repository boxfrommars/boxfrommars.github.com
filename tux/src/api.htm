API

Заказы

Получение заказа

Request

GET /order/:orderId

orderId Идентификатор заказа

Response

id int Идентификатор заказа
customer customer Информация о клиенте
areaId int Идентификатор группы городов. На её основе берутся настройки заказа применительные к данной группе, включая часовой пояс
rides [ride] Список поездок
HTTP/1.1 200 OK 
include "api/order.json"

Создание заказа

Request

POST /order

id Идентификатор заказа
customer Информация о клиенте
areaId Идентификатор группы городов. На её основе берутся настройки заказа применительные к данной группе, включая часовой пояс
rides Список поездок

Response

Ответ точно такой же, как и при получении заказа, то есть сформированный заказ, с указанной информацией о заказчике, с присвоенными идентификаторами поездок и прочее.

Создание поездки

Request

POST /order/:orderId/ride

orderId int Идентификатор заказа (нужен для того чтобы привязать его к клиенту, при этом, если указан другой номер для связи создаётся ещё один заказ, для указанного номера)
orderType string Тип вызова, возможные значения emergency, normal, preorder
preOrderTime ISO 8601 Время подачи
alertTypeId Способ оповещения, берётся из словаря способов оповещений
orderedProperties [] Список заказанных свойств
orderedProperties.options [int] Список идентификаторов опций автомобиля, берётся из словаря опций автомобиля
orderedProperties.type {} Тип автомобиля
orderedProperties.type.id int Тип автомобиля, словарь типов ТС
orderedProperties.type.classId int Класс автомобиля, если есть. Например у типа легковой автомобиль есть класс. Словарь классов ТС
phone string Номер для связи
description string Комментарий к поездке
departureAdress {} Пункт отбытия, см. адрес
destinationAdress {} Пункт назначения, см. адрес
vehicle {} Информация о транспортном средстве, см. траснпортное средство 
include "api/ride-post.json"

Response

Такой же как и при получении поездки, с указанным id, с проставленными id для адресов, с таймлайном, в котором есть событие "заказ поездки"

Редактирование поездки

Request

POST /ride/:rideId

rideId int Идентификатор поездки

Все параметры такие же (?), как и при создании поездки

Отмена поездки

Request

POST /ride/:rideId/cancel

rideId int Идентификатор поездки

Response

success true Всегда передаётся true, так как в случае ошибки используется поведение при ошибках

Поиск

Request

GET /search?page=1&per_page=20

page int Номер страницы результата поиска
per_page int Сколько найденных заказов отображается на одной странице результатов

Запрос на поиск. Все поля не обязательны, но хотя бы одно должно присутствовать

phone string Номер телефона
destination Пока не понятно что
departureAddress Пока не понятно что
startTimeRange [ISO 8601, ISO 8601] интервал, в котором началась поездка. пример: [2014-01-28T13:00:60Z, 2014-01-28T14:00:60Z]
endTimeRange [ISO 8601, ISO 8601] интервал, в котором поездка закончиалсь.
vehicle {} параметры поиска по ТС
vehicle.callSign string Позывной
vehicle.brand stringМарка ТС
vehicle.colorId int Идентификатор цвета, см. словарь цветов
vehicle.type {} Тип автомобиля
vehicle.type.id int Тип автомобиля, словарь типов ТС
vehicle.type.classId int Класс автомобиля, если есть. Например у типа легковой автомобиль есть класс. Словарь классов ТС
vehicle.number string Номер ТС 
include "api/search-request.json"

Response

total int Всего заказов удовлетворяющих поиску
items [order] Заказы, собственно результаты поиска
HTTP/1.1 200 OK 
include "api/search-response.json"
