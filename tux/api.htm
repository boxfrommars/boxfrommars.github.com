Создание заказа

Request

POST /order

id Идентификатор заказа
customer Информация о клиенте
areaId int Идентификатор группы городов. На основе данной группы берутся настройки заказа, применительные к данной группе, включая часовой пояс
rides Список поездок

Response

Ответ точно такой же, как и при получении заказа -- сформированный заказ, с указанной информацией о заказчике, с присвоенными идентификаторами поездок и прочее.

Редактирование поездки

Request

POST /ride/:rideId

rideId int Идентификатор поездки

Все параметры такие же (?), как и при создании поездки

Отмена поездки

Request

POST /ride/:rideId/cancel

rideId int Идентификатор поездки

Response

success true Всегда передаётся true, так как в случае ошибки используется поведение при ошибках

Таймлайн
type string Тип сообщения
time ISO 8601 Время подачи
message string Сообщение
data {} Дополнительные данные
Возможные типы сообщения
ride_order заказ поездки
vehicle_assign назначение автомобиля
ride_change изменение поездки
vehicle_arrive прибытие автомобиля
customer_not_go клиент не выходит
ride_cancel отмена поездки
ride_start начало поездки
ride_end окончание поездки
call звонок (если не входит в предыдущие пункты)
message сообщение от водителя (если не входит в предыдущие)

    "type": "arrive", // тип события, такие как:  звонок клиента (является ли это событием для лога?), назначение автомобиля (assign), автомобиль приехал, дозвон не удался, сообщение от пассажира и т.д.
    "time": "2014-01-28T13:21:60Z", // unix timestamp события

    "data": {} // какая-то доп. информация
