Создание заказа

Request

POST /order

id Идентификатор заказа
customer Информация о клиенте
areaId Идентификатор группы городов. На её основе берутся настройки заказа применительные к данной группе, включая часовой пояс
description Общее описание к заказу
rides Список поездок

Response

Ответ точно такой же, как и при получении заказа, то есть сформированный заказ, с указанной информацией о заказчике, с присвоенными идентификаторами поездок и прочее.

Адрес

Используется в поездках

id int id адреса, если адрес есть в базе
area {} Группа городов
area.id int Идентификатор группы городов
area.name string Название группы городов
city {} Город
city.id int Идентификатор города
city.name string Название города
street {} Улица
street.id int Идентификатор улицы
street.name string Название улицы
house string Дом
entrance string Подъезд
description string Описание
location {} Координаты адреса
location.lt WGS 84 Широта
location.ln WGS 84 Долгота

Поля *.id и *.name -- при запросе на сервер необязательны, достаточно только одного из них: id, если улица/город/группа существует в словаре; name если нет. В ответах сервера оба поля обязательны

Возможные типы сообщения
ride_order заказ поездки

customer_not_go клиент не выходит
ride_cancel отмена поездки
ride_start начало поездки
ride_end окончание поездки
call звонок (если не входит в предыдущие пункты)
message сообщение от водителя (если не входит в предыдущие)

    "type": "arrive", // тип события, такие как:  звонок клиента (является ли это событием для лога?), назначение автомобиля (assign), автомобиль приехал, дозвон не удался, сообщение от пассажира и т.д.
    "time": "2014-01-28T13:21:60Z", // unix timestamp события
